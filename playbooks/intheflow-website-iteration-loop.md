# InTheFlow Website Iteration Loop (How we keep it alive)

## Why this matters
A website is not "done" after launch. It either stays aligned with reality, or it drifts.

For InTheFlow, the best results came from short iteration loops:
- review real wording,
- reduce cognitive load,
- sharpen one clear next action,
- ship small improvements fast.

## What we changed recently
- Reduced text overload without killing personality
- Removed stale year-based references
- Kept events as recurring yearly formats
- Shifted contact flow to WhatsApp-first
- Added a dedicated privacy page
- Aligned structured data with actual contact strategy

## Practical operating mode
### 1) Weekly micro-review (15–20 min)
- Is the hero still true?
- Is the primary CTA still the right one?
- Are course/event details current?
- Any sentence that feels heavy or unclear?

### 2) Monthly trust review (30 min)
- Mobile readability check
- Contact path check (WhatsApp + fallback channels)
- Legal/footer links check
- Meta description/title still matching current offer

### 3) Release discipline
Every content change should answer:
1. What became clearer for visitors?
2. What action is now easier?
3. What was removed to reduce noise?

## Rule of thumb
**Keep warmth. Cut noise. Preserve direction.**

That combination converts better than either "pure marketing" or "pure biography".
